# The Spinoff, nz-statute-book, and PCO count different layers of legislative change

## Executive answer

- The Spinoff's 241,456 figure is an amendment-history event count across Acts and regulations since 1872, and its per-government "flat churn" claim uses amendment events to Acts since 2008, not whole-Act reprints and not policy reversals alone [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted).
- `jonnonz1/nz-statute-book`'s 15,282 figure is a consolidation/version count for 4,001 currently mirrored public Acts, where each electronic consolidation becomes one git commit that modifies exactly one Act file [nz-statute-book README, lines 3-9](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L3-L9) and each rendered line is a semantic unit so diffs show changed provisions [nz-statute-book README, lines 26-39](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L26-L39).
- PCO's current public data model is not a single "change count": the official API exposes works, versions, and formats, where a work can have one or more versions and each version can have source XML, PDF, or HTML formats [New Zealand Legislation developer API](https://www.legislation.govt.nz/developer-api/); the Whiplash parser then derives provision-level events from XML `<history-note>` blocks with fields for amended provision, operation, date, amending provision, and amending legislation [Whiplash `parse_history.py`, lines 1-19](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/parse_history.py#L1-L19).
- Measured in operation units, "flat churn" still looks broadly flat for Act amendments since 2008 in the available Whiplash data, but the repeal share varies by government and would be invisible in a consolidation-count view [Whiplash `churn-aggregates.json`, lines 5-127](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/public/data/churn-aggregates.json#L5-L127).
- The safest conclusion is that the three numbers are reconcilable as a hierarchy: PCO works/versions/XML are the source layer; nz-statute-book commits are one consolidation/version per public Act; The Spinoff/Whiplash counts are parsed amendment-operation events inside those versions [PCO API](https://www.legislation.govt.nz/developer-api/); [nz-statute-book README](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md); [Whiplash parser](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/parse_history.py).

**Overall confidence:** Medium - the denominators and code paths are well documented, but I could not independently re-run the multi-GB PCO XML mirror or directly fetch live XML in this environment because legislation.govt.nz returned bot-protection/HTML to both curl and the vendored `legislation-nz` skill.

## Evidence

### The Spinoff count is provision-event churn, not consolidation churn

The Spinoff article says John Gregoriadis downloaded "every act of parliament and every regulation made under one" and parsed "every recorded change", producing 241,456 changes with the oldest dating to 1872 [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted). The same article then narrows the per-government chart to "amendment events to acts - not the regulations made under them - per term since 2008", listing 13,426, 15,144, 14,584, 17,538, 16,818, and 12,710 Act amendment events for successive terms [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted).

The article also warns that this is not a reversal-only count: it says the measure is "every amendment, not just reversals" and includes consequential tidy-ups and routine maintenance [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted). That caveat matters because the companion Whiplash application separates project/policy reversals from statute-book churn: its README describes a map of reversed projects, a statute-book churn timeline, and a running total that counts only money actually spent [Whiplash README, lines 1-6](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/README.md#L1-L6).

The Whiplash front-end source repeats the 241,456 figure as "the Parliamentary Counsel Office's XML of the entire statute book, parsed into 241,456 amendment events" [Whiplash `Methodology.svelte`, lines 121-134](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/src/components/Methodology.svelte#L121-L134). That corroborates the Spinoff denominator as a parsed event count from PCO XML, not a count of reprints, Acts, or policy reversals [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted); [Whiplash methodology source](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/src/components/Methodology.svelte#L121-L134).

### nz-statute-book counts electronic consolidations of public Acts

`nz-statute-book` describes itself as "Every public Act of the New Zealand Parliament, every electronic consolidation, as a commit" and reports 4,001 Acts and 15,282 consolidations, current through consolidations as at 2026-06-06 [nz-statute-book README, lines 1-9](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L1-L9). The same README says each commit modifies exactly one file, the markdown renders one semantic unit per line, and the commit body lists the amendments effective that day [nz-statute-book README, lines 17-30](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L17-L30).

That makes the 15,282 denominator narrower than The Spinoff's 241,456 in at least three ways: it is public Acts only rather than Acts plus regulations; it is current through the electronic consolidation corpus represented in the repo rather than every parsed operation since 1872; and it has one commit per Act-version/reprint event, even when a single reprint body lists many amendment operations [nz-statute-book README, lines 3-9](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L3-L9); [nz-statute-book README, lines 26-39](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L26-L39). The repo explicitly says history notes and tables of contents are stripped from the rendered Act files so the git diffs show legislative text change, while the amendment graph remains in the Whiplash pipeline [nz-statute-book README, lines 32-39](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L32-L39).

### PCO's native public model is works, versions, formats, and XML history notes

The official New Zealand Legislation developer API says it provides structured data describing legislation records and links to HTML, PDF, and XML versions [New Zealand Legislation developer API, lines 73-76 as fetched by WebSearch](https://www.legislation.govt.nz/developer-api/). The API page defines a three-tier model: a work is a specific Act or Bill, a work has one or more versions such as an Act "as at" a date or a Bill at a parliamentary stage, and each version has multiple formats such as XML, PDF, or HTML [New Zealand Legislation developer API, lines 77-92 as fetched by WebSearch](https://www.legislation.govt.nz/developer-api/).

The same official API documentation says API access requires an API key requested from PCO and that the current API has `/v0/works`, `/v0/works/{work_id}/versions`, and `/v0/versions/{version_id}` endpoints [New Zealand Legislation developer API, lines 86-100 as fetched by WebSearch](https://www.legislation.govt.nz/developer-api/). A separate official data.govt.nz case study says new documents and new versions of existing documents are published on the legislation website, while PDF and XML versions are added to a subscriber website whose XML directory structure provides access to current and previous versions in XML and PDF formats through data.govt.nz [data.govt.nz PCO case study](https://data.govt.nz/catalogue-guide/showcase/parliamentary-counsel-office).

The live data.govt.nz catalogue page itself was blocked by Incapsula to curl and `scripts/fetch.mjs` in this environment, and `scripts/archive-cite.mjs` could only reuse a snapshot that also contained the challenge page, so I cite the live catalogue URL only as the intended catalogue entry and use the official data.govt.nz case study for the substantive bulk-XML claim [data.govt.nz catalogue live URL, blocked during this run](https://catalogue.data.govt.nz/dataset/new-zealand-legislation); [data.govt.nz PCO case study](https://data.govt.nz/catalogue-guide/showcase/parliamentary-counsel-office).

Whiplash's parser shows how the public XML becomes operation units: it parses `<history-note>` blocks containing `<amended-provision>`, `<amending-operation>`, `<amendment-date>`, `<amending-provision>`, and `<amending-leg>` into an events table [Whiplash `parse_history.py`, lines 1-19](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/parse_history.py#L1-L19). The parser normalizes free-text operation strings into `inserted`, `substituted`, `replaced`, `repealed`, `revoked`, `amended`, `editorial`, or `other`, with `omit` mapped to `repealed` [Whiplash `parse_history.py`, lines 50-70](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/parse_history.py#L50-L70).

### Operation units answer a different question from consolidation events

The Whiplash export code defines `CHURN_OPS` as `amended`, `inserted`, `substituted`, and `replaced`, and defines `DEAD_OPS` as `repealed` and `revoked` [Whiplash `export_churn.py`, lines 30-35](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/export_churn.py#L30-L35). Its aggregate export scans events dated from 2008 onward, buckets them by government, increments amendment counts for `CHURN_OPS`, increments repeal counts for `DEAD_OPS`, and separately keeps public-Act-only counts in `amendments_acts` and `repeals_acts` [Whiplash `export_churn.py`, lines 187-225](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/pipeline/export_churn.py#L187-L225).

The generated aggregate data matches The Spinoff's Act-amendment numbers: `amendments_acts` is 13,426 for Key 1, 15,144 for Key 2, 14,584 for Key 3, 17,538 for Ardern 1, 16,818 for Ardern 2, and 12,710 for the partial Luxon 1 term [Whiplash `churn-aggregates.json`, lines 5-127](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/public/data/churn-aggregates.json#L5-L127); [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted). The same data shows Act repeal/revocation event counts of 2,391, 3,499, 2,844, 2,467, 2,648, and 2,022 for those same terms, with Clark 3 and Luxon 1 marked partial because the scan starts in 2008 and the current term is not complete [Whiplash `churn-aggregates.json`, lines 5-127](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/public/data/churn-aggregates.json#L5-L127).

This means an operation-unit view would not overturn the article's basic "not red or blue" point for Act amendments, because the public-Act amendment counts stay within a relatively narrow band across complete post-2008 terms [The Spinoff, 2026-07-07](https://thespinoff.co.nz/politics/07-07-2026/are-we-really-the-fastest-repealers-in-the-west-i-counted); [Whiplash `churn-aggregates.json`, lines 5-127](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/public/data/churn-aggregates.json#L5-L127). It would, however, expose a different shape: repeal/revocation events as a share of Act amendment-plus-repeal events range from about 12.3% in Ardern 1 to about 18.8% in Key 2 in the generated post-2008 data, so a repeal-share question cannot be answered from either The Spinoff headline total or the nz-statute-book consolidation count alone [Whiplash `churn-aggregates.json`, lines 5-127](https://github.com/jonnonz1/whiplash/blob/420d2845d32c38491f3ed603eebedd2ada73572a/public/data/churn-aggregates.json#L5-L127).

## What would change this conclusion

- A successful independent re-run of the full PCO XML mirror and Whiplash parser could raise confidence from Medium to High, especially if it reproduced the 241,456 event total and the per-term aggregate data from clean inputs.
- Direct access to PCO's live XML in this environment would let the finding cite raw `<history-note>` examples rather than the Whiplash parser's documentation and code; curl and the `legislation-nz` skill both failed with bot-protection/HTML during this run.
- A newer public PCO API release that exposes amendment-history events directly, rather than only works/versions/formats and linked XML, would change the "PCO has no single native change count" conclusion.
- I could not verify whether `nz-statute-book` excludes all repealed public Acts from the 4,001-Act denominator or includes some no-longer-in-force Acts that still appear in the generated repo; the README says "every public Act" and "current through consolidations", while the issue text says "currently in-force public Acts only" [nz-statute-book README, lines 3-9](https://github.com/jonnonz1/nz-statute-book/blob/1273f588169084689c052a1c685f2a7ec1481720/README.md#L3-L9).

# Public NZ grants data is mostly awards history, not an eligibility-ready opportunities feed

## Executive answer

- As at 2 July 2026, I found no single public official New Zealand feed that exposes current grant opportunities with the fields needed for automated eligibility matching: eligibility, exclusions, geography, deadline, legal-status rules, amount range, application link and update cadence. The best official machine-readable sources I found are award/recipient datasets and charity-register data, not opportunity records; the audited sources cover central-government/statutory examples, one council example and two private products, not the whole funding landscape. Centre for Social Impact describes eight main funding-source types in Aotearoa New Zealand, each with its own grant criteria, priorities and processes, so this audit should not be read as comprehensive coverage of gaming trusts, philanthropic foundations, licensing trusts or every council. [DIA class 4 grants data](https://www.dia.govt.nz/Gambling-statistics-class-4-grants-data); [Charities Services open data](https://www.charities.govt.nz/charities-in-new-zealand/the-charities-register/open-data); [data.govt.nz Lottery Grants Board Community Committee grant recipients](https://catalogue.data.govt.nz/dataset/lottery-grants-board-community-committee-grant-recipients); [Centre for Social Impact funding guide](https://www.centreforsocialimpact.org.nz/knowledge-base/funding-guide)
- DIA's class 4 grants data is the strongest official structured source I found for grants distribution: DIA says it publishes class 4 grants and applied-funds data through [Granted.govt.nz](https://www.granted.govt.nz/) and an Excel dataset, while the field list I inspected is from DIA's document explicitly labelled "Draft guidance for submitting class 4 grants data", not from a parsed copy of the live Excel workbook. That draft guidance defines proposed fields such as applicant organisation, NZBN, charities registration number, categories, benefiting territorial authority, amount requested, received date, approval/decline date, reason for decline and amount granted. [DIA class 4 grants data](https://www.dia.govt.nz/Gambling-statistics-class-4-grants-data); [DIA guidance for submitting class 4 grants data](https://www.dia.govt.nz/diawebsite.nsf/Files/Gambling-2025/%24file/Guidance-for-submitting-class-4-grants-data.pdf)
- Charities Services open data is useful for applicant-side matching because the OData service root and metadata expose charity-register, officer and annual-return entities, and direct unauthenticated endpoint requests return ATOM by default plus JSON and CSV when requested. Those endpoints describe charities/register data, not a grants-opportunities catalogue. [Charities Services OData service root](http://www.odata.charities.govt.nz/); [OData metadata](http://www.odata.charities.govt.nz/$metadata); [example JSON endpoint](http://www.odata.charities.govt.nz/Organisations?$top=1&$format=json); [example CSV endpoint](http://www.odata.charities.govt.nz/Organisations?$top=1&$format=csv)
- Community Matters has current fund pages with eligibility, priorities, exclusions, opening dates and application links in human-readable webpages, and data.govt.nz has older CSV/XLSX recipient datasets for some Lottery/COGS grants, but I did not find a current machine-readable Community Matters opportunities API or download that combines those fields. [Community Matters COGS](https://www.communitymatters.govt.nz/community-organisations-grants-scheme); [Community Matters Lottery Community](https://www.communitymatters.govt.nz/lottery-community); [data.govt.nz Lottery Community Sector Research grant recipients](https://catalogue.data.govt.nz/dataset/lottery-grants-board-community-sector-research-grant-recipients)
- A useful MVP could use public data for funder discovery, applicant enrichment and historical-grant signals, but not for reliable eligibility matching without manual assembly of current opportunity records from official webpages and funder websites. Private/product databases such as Fundsorter and Generosity NZ appear to have solved parts of this data assembly commercially, but they are not public official datasets. [Fundsorter](https://www.fundsorter.com/); [Generosity NZ](https://kohaa.org.nz/); [Creative NZ other funding sources](https://creativenz.govt.nz/funding-and-support/all-opportunities/other-sources-of-funding-and-support)

**Overall confidence:** Medium - the core conclusion is supported by official source inventories and field checks, but data.govt.nz blocks some direct automated fetches with Incapsula, Community Matters blocks plain HTTP requests with a Cloudflare challenge while loading in browser-based checks, and I did not exhaustively inspect every council, gaming trust and philanthropic funder website.

### What public data can support in an MVP

Public official data can support three MVP functions. First, Charities Services can enrich applicant profiles with charity registration, status, activity, beneficiary, sector, area-of-operation, NZBN and annual-return information. [Charities Services open data](https://www.charities.govt.nz/charities-in-new-zealand/the-charities-register/open-data)

Second, DIA class 4 and public recipient datasets can provide historical evidence about who has been funded, by whom, for what categories, in what places and at what amounts. [DIA class 4 grants data](https://www.dia.govt.nz/Gambling-statistics-class-4-grants-data); [DIA guidance](https://www.dia.govt.nz/diawebsite.nsf/Files/Gambling-2025/%24file/Guidance-for-submitting-class-4-grants-data.pdf); [data.govt.nz WCC Funding Open Data](https://catalogue.data.govt.nz/dataset/wcc-funding-open-data)

Third, official webpages can seed an opportunities catalogue when a human or controlled scraper extracts fields from the page. COGS and Lottery Community pages already contain the relevant rules in text, including dates/status, eligible organisation types, priority groups, documents, fundable activities and exclusions. [Community Matters COGS](https://www.communitymatters.govt.nz/community-organisations-grants-scheme); [Community Matters Lottery Community](https://www.communitymatters.govt.nz/lottery-community)

Public data alone is not enough for a reliable eligibility-matching MVP unless the project creates and maintains a new opportunities schema. The minimum manual schema should include funder, fund name, status, opening date, closing date, decision date, application URL, geography, eligible legal statuses, charity-registration requirement, informal-group/fundholder rules, amount or term limits, eligible costs, exclusions, priority groups, required documents, reporting obligations, source URL, last checked date and update cadence. This conclusion follows from the observed gap between award/applicant datasets and current fund-rule webpages, and from independent sector evidence that funding-source criteria are distributed across many source types rather than a single public data feed. [Charities Services OData metadata](http://www.odata.charities.govt.nz/$metadata); [DIA class 4 grants data](https://www.dia.govt.nz/Gambling-statistics-class-4-grants-data); [Community Matters COGS](https://www.communitymatters.govt.nz/community-organisations-grants-scheme); [Creative NZ other funding sources](https://creativenz.govt.nz/funding-and-support/all-opportunities/other-sources-of-funding-and-support); [Centre for Social Impact funding guide](https://www.centreforsocialimpact.org.nz/knowledge-base/funding-guide)

## What would change this conclusion

- A current official API or bulk download for Community Matters funding opportunities, with normalized fields for eligibility, exclusions, deadlines, geography, amount ranges and application links, would change the recommendation.
- A data.govt.nz catalogue/API route that reliably exposes current opportunity records across DIA, councils, gaming trusts and statutory funders would change the conclusion that manual assembly is required.
- Product access or a data-sharing agreement with Fundsorter, Generosity NZ, or another grants database could change the build path, but it would not change the conclusion about public official data unless reuse rights allowed public/open use.
- I could not verify every data.govt.nz resource directly because several catalogue/API fetches returned Incapsula incident pages during automated access; I used rendered search metadata, successful rendered pages and official source pages where possible. Community Matters pages also returned Cloudflare 403 challenges to plain HTTP requests during review, although browser-based checks loaded them.
- I did not audit every council or every class 4 society website, so there may be additional local machine-readable datasets beyond the Wellington City Council example.
- I did not parse the live DIA class 4 Excel workbook from data.govt.nz in this revision; the detailed field list remains tied to DIA's draft guidance, so the exact published workbook columns remain a follow-up check.
- A human funder, grants adviser or community-organisation user should review the proposed manual schema before build work, because eligibility language often contains judgement calls that are hard to encode from webpages alone.

## Open follow-up questions

- Which councils besides Wellington publish grants application/outcome data in machine-readable formats, and do any publish current opportunity rules as data?
- Can Community Matters records of grants be converted into a stable, normalized historical-awards dataset by fund and year without violating site terms or creating maintenance risk?
- Which private/product database, if any, would allow low-cost or public-benefit reuse of structured NZ grant opportunities?
- What exact eligibility schema gives the best tradeoff between usefulness for small charities and maintainability by volunteers?

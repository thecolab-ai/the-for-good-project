Full-text XML pull for classification
-------------------------------------

Commands (the `latest.xml` endpoint; use curl directly to a file for the two
larger Acts so the whole document is captured):

```bash
node scripts/fetch.mjs "https://www.legislation.govt.nz/act/public/2025/72/en/latest.xml/"   # 32,658 bytes, complete

```

fetch.mjs reported `# fetched via plain HTTP` and returned real legislation XML
(root `<act ...>`), not the AWS WAF challenge body seen on 7 July. No
`x-amzn-waf-action: challenge` or `window.gokuProps` appeared in any response.
`fetch.mjs` truncates its stdout at MAX_CHARS (default 20,000; raise via env), so
for the two larger Acts the full document was captured with a direct `curl` to a
file — a display cap, not an access block.

Classification method
---------------------

Each Act's XML was parsed with Python's xml.etree.ElementTree. Top-level amending
provisions were taken as `<prov>` elements with no `<amend>` or `<schedule>`
ancestor (this excludes the inserted/replaced content blocks, which live inside
`<amend>`). Title, Commencement, and "Principal Act/regulations/rules" locator
provisions were excluded. For each remaining provision the operative instruction
text (all text not inside an `<amend>` block) was scanned for the operation verbs
insert / replace / repeal / revoke / omit / delete; a provision spanning more than
one family is `mixed`. Running this parser against Act 2025/85 reproduced the exact
15 insert / 5 replace / 1 repeal-revoke / 2 mixed split recorded by the earlier
browser-backed classification of that Act, which validates the method. Full
per-provision results are embedded in statutory_amendment_sample.py.
